# WealthCopy 90초 데모 스크립트

목표 러닝타임: **1분 30초**. 메인 화면에는 L7, 행동 세 개, 행동 완료율만 보여 준다.

## 0:00–0:12 — 문제 재정의

화면: 초기 WealthCopy 화면의 `다음 자산 단계 L7`.

멘트: “대부분의 자산관리 서비스는 더 많은 분석을 보여 줍니다. 하지만 문제는 정보 부족이 아니라, 알고도 실행하지 못하는 것입니다.”

## 0:12–0:27 — 행동 만들기

동작: `L7 행동 복제하기`를 누른다. 미리 채운 조건과 “내년에 이사 계획이 있어 현금 여유를 유지하고 싶다”를 짧게 보여 주고 `3가지 행동 복제하기`를 누른다.

멘트: “조건은 행동을 만드는 동안에만 사용되고 메인 화면에는 남지 않습니다.”

## 0:27–0:43 — 단순한 결과

화면: `L7`, 이번 달 행동 정확히 세 개, `0 / 3 완료 · 0%`.

멘트: “사용자가 보는 것은 다음 단계, 지금 할 행동 세 개, 완료율뿐입니다. 경로 비교와 추천 이유, 금액과 기간은 보여 주지 않습니다.”

## 0:43–0:57 — GPT‑5.6의 내부 역할

화면: 제품 화면을 유지하거나 공개 API 응답 JSON을 짧게 오버레이한다.

멘트: “서버 안에서 GPT‑5.6은 자연어 변화를 해석해 허용된 행동 ID만 선택합니다. 성공 응답도 L7, 세 행동, 진행률 세 키뿐이고 모델 설명은 브라우저로 나오지 않습니다.”

확인할 JSON: `{nextLevel, actions, progress}`. 각 action은 `{id, completed}`만 가진다.

## 0:57–1:17 — 행동과 완료율

동작: 첫 행동을 체크해 `1 / 3 · 33%`, 두 번째를 체크해 `2 / 3 · 67%`, 마지막을 체크해 `3 / 3 · 100%`를 보여 준다.

멘트: “완료율은 오직 세 행동의 체크 상태입니다. 자산 수익이나 L7 도달률을 뜻하지 않습니다.”

## 1:17–1:30 — 마무리

화면: 세 행동 완료 상태와 교육용 안내.

멘트: “키나 모델 연결이 없어도 같은 형태의 fallback이 행동을 준비합니다. WealthCopy는 더 많이 분석하는 서비스가 아니라, 이번 달 행동을 시작하게 하는 서비스입니다.”

## 녹화 전 체크리스트

- [ ] `pnpm.cmd check` 통과
- [ ] `pnpm.cmd dev` 후 `http://localhost:3000` 접속
- [ ] `POST /api/v2/plan` 성공 응답 최상위가 정확히 `nextLevel`, `actions`, `progress`
- [ ] actions가 정확히 3개이고 각 객체가 `id`, `completed`만 포함
- [ ] progress가 `0 → 33 → 67 → 100`으로 변경
- [ ] 화면에 경로 비교, 추천, 금액, 기간, 모델명, fallback 출처가 없는지 확인
- [ ] 새 녹화 전 `wealthcopy-public-plan-v2`와 필요 시 `wealthcopy-demo-plan-v1` 삭제
- [ ] API 키, 개발자도구, 개인 데이터가 녹화에 노출되지 않도록 정리
