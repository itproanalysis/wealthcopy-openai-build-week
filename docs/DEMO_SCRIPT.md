# WealthCopy 90초 데모 스크립트

목표 러닝타임: **1분 30초**. 메인 화면에는 현재 단계에서 이어지는 다음 단계, 행동 세 개, 행동 완료율만 보여 준다.

## 0:00–0:12 — 문제 재정의

화면: 초기 WealthCopy 화면의 중립적인 `다음 자산 단계 NEXT`.

멘트: “대부분의 자산관리 서비스는 더 많은 분석을 보여 줍니다. 하지만 문제는 정보 부족이 아니라, 알고도 실행하지 못하는 것입니다.”

## 0:12–0:27 — 행동 만들기

동작: `다음 단계 행동 3개 복제하기`를 누른다. 자가 선택 현재 단계 `L3 · 현금 안전망`, `소득 대비 48%`, `자가 선택 PSID 참고 구간 · 참조 분포 50–74백분위`, `부채비율 18%`를 짧게 보여 주고 행동 복제 버튼을 누른다.

멘트: “L7은 하나의 예시였습니다. 이제 시작, 흐름 정리, 현금 안전망부터 장기 유지까지 L1–L7 전체 여정이 연결됩니다. 단계는 사용자가 스스로 고르는 실행 여정이고, PSID 구간은 미국 공개 집계의 경계 구조를 빌린 개념형 참고 구간입니다. 둘 다 한국의 공식 자산등급이나 검증된 순위가 아닙니다.”

## 0:27–0:43 — 단순한 결과

화면: `L4`, 이번 달 행동 정확히 세 개, `0 / 3 완료 · 0%`.

멘트: “사용자가 보는 것은 다음 단계, 지금 할 행동 세 개, 완료율뿐입니다. 경로 비교와 추천 이유, 금액과 기간은 보여 주지 않습니다.”

## 0:43–0:57 — GPT‑5.6의 내부 역할

화면: 제품 화면을 유지하거나 공개 API 응답 JSON을 짧게 오버레이한다.

멘트: “서버는 현재 단계의 전환 앵커를 먼저 보존합니다. GPT‑5.6은 그 전환에 허용된 안전한 동반 행동만 선택하고, 안전 제약이 있으면 규칙이 우선합니다. 마지막 행동은 항상 월말 점검입니다. 성공 응답도 다음 단계, 세 행동, 진행률 세 키뿐입니다.”

확인할 JSON: `{nextLevel, actions, progress}`. 각 action은 `{id, completed}`만 가진다.

## 0:57–1:15 — 행동과 실제 캘린더 연결

동작: 첫 행동을 체크해 `1 / 3 · 33%`, 두 번째를 체크해 `2 / 3 · 67%`를 보여 준다. 월말 점검 카드에서 `월말 일정 파일 받기`를 눌러 개인정보가 없는 `.ics` 파일 연결을 보여 준 뒤 마지막 행동을 체크해 `3 / 3 · 100%`로 만든다.

멘트: “월말 점검은 별도 계좌 연결 없이 캘린더로 옮길 수 있습니다. 완료율은 오직 세 행동의 체크 상태이며 자산 수익이나 다음 단계 도달률을 뜻하지 않습니다.”

## 1:15–1:30 — 상태 재확인과 마무리

동작: `다음 단계 이어가기`를 눌러 `L4 · 납부 안정`이 현재 단계 후보로 불러와진 설정 화면을 짧게 보여 준다.

멘트: “세 행동을 끝냈다고 자산 단계가 자동으로 오르지는 않습니다. 현재 상태를 다시 확인한 뒤 다음 전환을 이어 갑니다. WealthCopy는 더 많이 분석하는 서비스가 아니라, 행동을 복제하고 실행하고 완료하게 하는 서비스입니다.”

## 녹화 전 체크리스트

- [ ] `pnpm.cmd check` 통과
- [ ] `pnpm.cmd dev` 후 `http://localhost:3000` 접속
- [ ] `POST /api/v2/plan` 성공 응답 최상위가 정확히 `nextLevel`, `actions`, `progress`
- [ ] actions가 정확히 3개이고 각 객체가 `id`, `completed`만 포함
- [ ] progress가 `0 → 33 → 67 → 100`으로 변경
- [ ] `3/3` 완료 뒤 자동 승급하지 않고 이전 `nextLevel`을 현재 단계 후보로 제안해 재확인하는지 확인
- [ ] 월말 점검의 `.ics` 파일에 금융 입력이나 개인정보가 없는지 확인
- [ ] 화면에 경로 비교, 추천, 금액, 기간, 모델명, fallback 출처가 없는지 확인
- [ ] 설정에 원화·달러 금액이 없고 PSID 구간을 한국 실제 순위나 수익 예측으로 표현하지 않는지 확인
- [ ] 새 녹화 전 `wealthcopy-public-plan-v3`, 이전 `wealthcopy-public-plan-v2`, 필요 시 `wealthcopy-demo-plan-v1` 삭제
- [ ] API 키, 개발자도구, 개인 데이터가 녹화에 노출되지 않도록 정리
