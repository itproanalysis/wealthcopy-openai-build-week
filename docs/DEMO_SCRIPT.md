# WealthCopy 90초 데모 스크립트

목표 길이는 **1분 30초**입니다. 메인 화면에는 끝까지 다음 자산 단계, 이번 달 행동 3개, 행동 완료율만 남깁니다.

## 녹화 전 준비

- 기본 시나리오: 가구 총자산 `6.2억원`, 총부채 `0.7억원`, 소득 대비 실행 비율 `35%`, 부채상환 비율 `15%`, PSID 참고 구간 `잘 모르겠어요`, 선택 메모 없음
- 계산 확인: 가구 순자산 `5.5억원` → 서버 분류 `L7` → 공개 다음 단계 `L8`
- 경계 시나리오: 별도 데모 탭에서 총자산 `10,000억원`, 총부채 `0원` → 순자산 1조원 → `L15 → L15` 유지
- 실제 개인정보나 실제 가구 금액은 사용하지 않습니다.
- API 키, 요청 원문, 브라우저 확장 프로그램 알림이 녹화에 보이지 않게 정리합니다.

## 0:00–0:10 — 문제를 행동으로 바꾸기

화면: 초기 메인 화면의 `NEXT`, 행동 3개 자리, `0%` 진행률을 보여 줍니다.

멘트: “기존 자산관리는 사용자가 분석하고 비교하고 판단하게 합니다. WealthCopy는 자산관리를 행동의 문제로 바꿉니다. 사용자가 보는 것은 어디로 가는지, 무엇을 할지, 얼마나 실행했는지뿐입니다.”

## 0:10–0:28 — 최신 가구 스냅샷으로 자동분류

동작: `시작하기`를 누르고 기본 시나리오 값을 입력합니다. PSID 참고 구간은 `잘 모르겠어요`로 둔 뒤 `레벨 분류하고 행동 3개 복제`를 누릅니다.

멘트: “가구 총자산에서 총부채를 뺀 순자산으로 서버가 현재 레벨을 자동분류합니다. 이 예시는 5억 5천만원이라 L7이고 다음 단계는 L8입니다. PSID 구간은 미국 공개 집계 구조를 빌린 선택 참고값일 뿐, 이 레벨 계산에는 쓰지 않습니다.”

## 0:28–0:43 — 결과는 세 가지뿐

화면: `L8`, 정확히 3개의 행동, `0 / 3 · 0%`를 한 화면에 보여 줍니다. 첫 행동은 L7 전환 앵커인 `지난 계획과 실행 차이 확인하기`, 세 번째 행동은 `월말 점검 일정 잡기`인지 확인합니다.

멘트: “복잡한 분류와 행동 구성은 뒤에서 끝납니다. 앞에는 L8, 이번 달 행동 세 개, 완료율만 남습니다. 경로 비교, 추천 이유, 금액, 기대수익률, 모델 이름은 보여 주지 않습니다.”

## 0:43–0:56 — GPT‑5.6의 제한된 역할

화면: 제품 화면을 유지하거나, 사전에 만든 짧은 아키텍처 자막을 겹칩니다. 요청 원문이나 개발자 도구는 열지 않습니다.

멘트: “레벨, 단계별 필수 행동, 안전 규칙은 서버가 결정합니다. GPT‑5.6은 정확한 자산·부채 금액이나 레벨을 보지 않고, 검토된 행동 ID와 통화 중립 비율 신호 안에서 동반 행동만 선택합니다. 성공 JSON도 `nextLevel`, `actions`, `progress` 세 키뿐입니다.”

자막 선택: `금액으로 서버 분류 → 금액 제거 → 허용 행동 안에서 AI 선택 → 3키 공개 계획`

## 0:56–1:15 — 실행, 완료율, 캘린더 연결

동작: 첫 행동을 체크해 `33%`, 두 번째 행동을 체크해 `67%`를 보여 줍니다. `월말 일정 파일 받기`를 누른 뒤 세 번째 행동을 체크해 `3 / 3 · 100%`로 만듭니다.

멘트: “체크할 때마다 행동 완료율만 33, 67, 100으로 바뀝니다. 월말 점검은 개인정보와 금융정보가 없는 일정 파일로 캘린더에 연결됩니다. 100퍼센트는 행동 세 개를 완료했다는 뜻이지, 자산이 늘었거나 단계에 도달했다는 뜻이 아닙니다.”

## 1:15–1:25 — 자동 승급 대신 새 스냅샷

동작: 완료 안내의 `최신 자산정보로 다시 분류`를 눌러 입력 화면을 다시 보여 줍니다.

멘트: “그래서 3/3이어도 레벨은 자동으로 오르지 않습니다. 새 달에는 최신 총자산과 총부채를 다시 입력하고 새 스냅샷으로 재분류합니다.”

## 1:25–1:30 — L15 유지와 마무리

화면: 미리 준비한 경계 시나리오 탭의 `L15 유지 단계`를 짧게 보여 줍니다.

멘트: “이 기준은 음의 순자산 L1부터 1조원 이상 L15까지 이어지며, L15는 존재하지 않는 상위 등급 대신 유지·운영 행동을 제공합니다. WealthCopy는 복제하고, 실행하고, 완료하게 합니다.”

## 녹화 전 검증 체크리스트

- [ ] `pnpm.cmd check` 통과
- [ ] `pnpm.cmd dev` 후 `http://localhost:3000` 접속
- [ ] 총자산 6.2억원·총부채 0.7억원이 `L7 → L8`로 분류됨
- [ ] 순자산 정확히 1조원이 `L15 → L15` 유지로 분류됨
- [ ] `POST /api/v2/plan` 성공 JSON 본문의 최상위 키가 정확히 `nextLevel`, `actions`, `progress`
- [ ] 성공 응답의 `X-WealthCopy-Source-Level` 헤더가 서버 분류 출발 레벨이고 `Cache-Control: no-store`
- [ ] actions가 정확히 3개이고 각 객체가 `id`, `completed`만 포함하며 세 번째가 `schedule_monthly_checkin`
- [ ] progress가 완료 수에 따라 `0 → 33 → 67 → 100`으로만 바뀜
- [ ] `3/3` 완료가 자산 도달률이나 자동 승급으로 표현되지 않고 최신 스냅샷 재분류로 연결됨
- [ ] 메인 화면에 다음 단계, 행동 3개, 완료율 외의 경로 비교·추천 이유·금액·기간·수익률·모델명이 없음
- [ ] 정확한 총자산·총부채가 요청 중 분류 외에 모델 입력, 성공 JSON, localStorage, ICS, 분석 이벤트 또는 로그에 없음
- [ ] `wealthcopy-public-plan-v4` 값이 정확히 `{version, monthKey, sourceLevel, plan}`이고 금액·비율·PSID·메모가 없음
- [ ] 익명 session UUID가 별도 `wealthcopy-anonymous-session` 키에 있으며 v4 계획 레코드에 섞이지 않음
- [ ] 과거 저장 키는 새 레벨이나 완료 상태로 변환되지 않고 제거됨
- [ ] 월이 바뀐 레코드는 폐기되고 최신 가구 자산정보 입력을 요구함
- [ ] PSID 선택을 바꾸거나 `unknown`으로 두어도 L1–L15 분류가 달라지지 않음
- [ ] `.ics` 파일에 금액, 레벨, PSID, 프로필, 행동 완료 또는 모델 데이터가 없음
- [ ] 녹화에 실제 개인정보, 실제 금융정보, API 키, 전체 요청 원문이 노출되지 않음

## 모델 실패 시 복구 멘트

“모델 연결이 없거나 출력이 검증을 통과하지 못해도 서버의 안전한 규칙 fallback이 같은 세 키 계약과 검토된 행동 세 개를 반환합니다. 사용자 화면에는 내부 공급자 상태가 노출되지 않습니다.”
