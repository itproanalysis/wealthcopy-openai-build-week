# WealthCopy 90초 데모 스크립트

목표 길이는 **1분 30초**입니다. 메인 화면에는 끝까지 다음 자산 단계, 이번 달 행동 3개, 행동 완료율만 남깁니다. 행동을 펼쳤을 때만 남는 결과, 완료 기준과 세 단계 체크리스트를 보여 줍니다.

## 녹화 전 준비

- 기본 시나리오 — 가구 자산: 총자산 `6.2억원`, 총부채 `0.7억원`
- 기본 시나리오 — 자산 구조: 가장 큰 범주 `부동산`, 한 범주 비중 `70% 이상`, 생활비 여유 `6–12개월`
- 기본 시나리오 — 실행 여건: 저축·상환 `35%`, 부채상환 `15%`, 소득 안정성 `잘 모르겠어요`, 부채 신호 `현재 특이사항 없음`, 90일 변화 `없음`, PSID `잘 모르겠어요`, 메모 없음
- 계산 확인: 가구 순자산 `5.5억원` → 서버 분류 `L7` → 공개 다음 단계 `L8`
- 행동 확인: `pause_dominant_bucket_additions` → `align_valuation_dates` → `verify_valuation_freshness`
- 경계 시나리오: 별도 데모 탭에서 총자산 `10,000억원`, 총부채 `0원` → 순자산 1조원 → `L15 → L15` 유지
- 모든 값은 데모용 가상값입니다. 실제 개인정보나 실제 가구 금액은 사용하지 않습니다.
- API 키, 요청 원문, 브라우저 확장 프로그램 알림이 녹화에 보이지 않게 정리합니다.

## 0:00–0:10 — 문제를 행동으로 바꾸기

화면: 초기 메인 화면과 `내 경로 시작하기`를 보여 줍니다.

멘트: “기존 자산관리는 사용자가 분석하고 비교하고 판단하게 합니다. WealthCopy는 자산관리를 행동의 문제로 바꿉니다. 사용자가 보는 것은 어디로 가는지, 무엇을 할지, 얼마나 실행했는지뿐입니다.”

## 0:10–0:32 — 3단계 수동 스냅샷

동작: `내 경로 시작하기`를 누르고 `가구 자산 → 자산 구조 → 실행 여건`을 순서대로 입력합니다. PSID 참고 구간은 `잘 모르겠어요`로 둡니다.

멘트: “현재 버전은 계좌 연결이 아니라 세 단계 수동 스냅샷입니다. 총자산과 총부채로 레벨을 분류하고, 가장 큰 자산 범주, 쏠림, 현금 여유, 소득과 부채 신호로 이번 달 병목을 정합니다. 정확한 금액은 서버 경계에서 레벨과 거친 레버리지 신호로 축약된 뒤 모델과 저장소에서 사라집니다.”

## 0:32–0:47 — 결과는 protect·advance·verify

화면: `L8`, `0 / 3 · 0%`, 세 행동의 단계 라벨을 한 화면에 보여 줍니다.

멘트: “이 가구는 L7에서 L8로 이어집니다. 행동은 쏠림을 먼저 보호하고, 모든 자산의 확인 기준일을 맞춘 뒤, 오래된 평가와 담당자를 검증하는 순서입니다. 내부 경로와 계산 이유는 숨기고 기반 보호, 다음 단계, 결과 확인만 남깁니다.”

화면에서 확인할 제목:

1. `쏠림 범주 추가 판단 멈추기`
2. `자산 확인 기준일 맞추기`
3. `자산 기준일 최신성 검증하기`

## 0:47–1:02 — 막연한 권고 대신 완료 가능한 체크리스트

동작: 두 번째 행동의 `실행 순서`를 펼쳐 `남는 결과`, `완료 기준`, 세 단계를 보여 주고 `체크리스트 복사`를 누릅니다.

멘트: “각 행동에는 무엇이 남아야 하는지, 언제 완료인지, 실제로 따라 할 세 단계가 있습니다. 체크리스트를 복사해 메모나 협업 도구에서 바로 실행할 수 있습니다. 상품과 거래를 추천하는 화면이 아니라 결과물이 남는 실행 도구입니다.”

## 1:02–1:14 — GPT‑5.6의 제한된 역할

화면: 제품 화면을 유지하거나 사전에 만든 짧은 아키텍처 자막을 겹칩니다. 요청 원문이나 개발자 도구는 열지 않습니다.

멘트: “레벨, 안전 중단선, 전환 행동과 검증 행동은 서버 정책이 고정합니다. hard stop이 없을 때 GPT‑5.6은 정확한 금액, 레벨, PSID를 보지 않고 검토된 후보 중 기반 보호 행동 하나만 선택합니다. 실패해도 같은 계약의 규칙 fallback이 작동합니다.”

자막 선택: `비공개 분류·밴드화 → 안전 후보 1개 선택 → 고정 전환·검증 결합 → 3키 공개 계획`

## 1:14–1:25 — 실행, 진행률, 재분류 루프

동작: 세 행동을 차례로 체크해 `33% → 67% → 100%`를 보여 주고 `최신 자산정보로 다시 분류` 버튼을 가리킵니다.

멘트: “체크할 때마다 행동 완료율만 바뀝니다. 100퍼센트는 세 행동을 완료했다는 뜻이지 자산이 늘었다는 뜻이 아닙니다. 새 달에는 최신 세 단계 스냅샷으로 레벨과 행동을 다시 분류합니다.”

## 1:25–1:30 — L15 유지와 마무리

화면: 미리 준비한 경계 시나리오 탭의 `L15 유지 단계`를 짧게 보여 줍니다.

멘트: “음의 순자산 L1부터 1조원 이상 L15까지 이어지고, L15는 존재하지 않는 상위 등급 대신 운영과 연속성을 점검합니다. WealthCopy는 복제하고, 실행하고, 완료하게 합니다.”

## 녹화 전 검증 체크리스트

- [ ] `pnpm.cmd check` 통과
- [ ] `pnpm.cmd dev` 후 `http://localhost:3000` 접속
- [ ] 입력이 `가구 자산 → 자산 구조 → 실행 여건`의 3단계이고 계좌 연결 또는 마이데이터 인증으로 표현되지 않음
- [ ] 총자산 6.2억원·총부채 0.7억원이 `L7 → L8`로 분류됨
- [ ] 기본 시나리오 행동이 `pause_dominant_bucket_additions`, `align_valuation_dates`, `verify_valuation_freshness` 순서임
- [ ] 순자산 정확히 1조원이 `L15 → L15` 유지로 분류됨
- [ ] `POST /api/v2/plan` 성공 JSON 최상위 키가 정확히 `nextLevel`, `actions`, `progress`
- [ ] 성공 응답의 `X-WealthCopy-Source-Level` 헤더가 서버 분류 출발 레벨이고 `Cache-Control: no-store`
- [ ] actions가 정확히 3개이고 각 객체가 `id`, `completed`만 포함함
- [ ] 세 행동의 정적 메타데이터가 `protect → advance → verify`이며 각각 남는 결과, 완료 기준, 세 단계 실행 순서를 가짐
- [ ] 체크리스트 복사 결과에 행동 제목, 남는 결과, 완료 기준과 세 단계가 포함되고 금융 프로필은 포함되지 않음
- [ ] progress가 완료 수에 따라 `0 → 33 → 67 → 100`으로만 바뀜
- [ ] `3/3` 완료가 자산 도달률이나 자동 승급으로 표현되지 않고 최신 스냅샷 재분류로 연결됨
- [ ] 메인 화면에 다음 단계, 행동 3개, 완료율 외의 경로 비교·추천 이유·금액·기간·수익률·모델명이 없음
- [ ] 정확한 총자산·총부채와 원 비율이 비공개 분류·밴드화 이후 모델 입력, 성공 JSON, localStorage, 분석 이벤트 또는 로그에 없음
- [ ] `wealthcopy-public-plan-v5` 값이 정확히 `{version, monthKey, sourceLevel, plan}`이고 `version: 5`이며 금액·비율·구조 신호·PSID·메모가 없음
- [ ] 익명 session UUID가 별도 `wealthcopy-anonymous-session` 키에 있으며 v5 계획 레코드에 섞이지 않음
- [ ] v4 이하 과거 저장 키는 새 행동이나 완료 상태로 변환되지 않고 제거됨
- [ ] 월이 바뀐 레코드는 폐기되고 최신 가구 자산정보 입력을 요구함
- [ ] PSID 선택을 바꾸거나 `unknown`으로 두어도 L1–L15 분류, 목적 경로, 행동 세 개가 달라지지 않음
- [ ] OpenAI 모델 입력에 PSID 선택값, 달러 기준값, 정확한 금액, 현재·다음 레벨과 원문 메모가 없음
- [ ] 녹화에 실제 개인정보, 실제 금융정보, API 키, 전체 요청 원문이 노출되지 않음

## 모델 실패 시 복구 멘트

“모델 연결이 없거나 출력 검증을 통과하지 못해도 서버의 안전한 규칙 fallback이 같은 세 키 계약과 `protect → advance → verify` 행동을 반환합니다. 사용자 화면에는 내부 공급자 상태가 노출되지 않습니다.”
